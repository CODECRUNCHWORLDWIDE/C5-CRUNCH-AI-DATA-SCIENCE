# Week 2 — Challenges

One challenge this week. Optional, but it is the realistic version of half the work you will ever do in a data role: joining three tables that almost-but-not-quite agree on keys.

1. **[Challenge 1 — Merge three tables](challenge-01-merge-three-tables.md)** — three small CSVs (orders, customers, products) with the realistic key bugs of real-world data: duplicate IDs, leading whitespace, inconsistent casing, and a row that has no matching customer. Produce one clean, validated, fully-joined table. (~2 hours)

This challenge is the technical foundation of the merge work you will do in any real EDA. Doing it carefully now means the mini-project's joins (and Week 4's feature engineering) feel like an extension of the same habit.
